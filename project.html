Link Prediction

We performed link prediction on subsets of the existing dataset. First, we chose a subset to work on after which we removed the citation edges that we wished to predict. This task allowed us to create a model that would predict the if a paper cited a previous paper or not. Below is the pipeline we used for the task.

We worked with knowledge graphs having varying amount of details to extract the embeddings.
  • A knowledge graph with only citation links
  • A knowledge graph with citation links, journals and authors
  • A knowledge graph with citation links, journals and authors and more detail on the authors, papers and journals themselves

We achieved the best results considering citation edges, authors and journals. It was observed that adding more information into this data did not help improve the accuracy and even reduced it by 1-2% for the test accuracy and by almost 5% for mAP.

We can see that 10 embeddings work well enough for the task.

We then moved on to the task of link prediction. We tried both machine learning and deep learning models and compared them.
For the machine learning models we used SVM, Logistic regression, Naive Bayes and Decision Trees.
For the deep learning models, we used a Dense Neural Network and a Graph Convolution Network.
All the machine learning models and the Dense Neural Network, we used the extracted embeddings as is from above. But for the Graph Convolution Network, we first create a neighbourhood graph, then use GCN to get the new relearned embeddings which are then passed to a linear layer with a softmax activation to classify the citation.
We could use a second order neighbourhood (that is a neighbourhood graph containing the second most closest nodes), but with the current level of optimization, it would take over a 100 days to finish one epoch! So instead we use the first order neighbourhood graph as the input for our GCN.

Lessons Learned

  • 1. Understanding Data
    • Nature of citation growth is exponential. Number of publications also increases exponentially.
  • 2. Addressing the Temporal nature of the data
    • Models like Logistic Regression seems to perform poorly, on test data, this could be due to the fact that they do not considers intermediate features, from which the algorithm actually extracts growth information.
  • 3. Anomalous Citaions
    • Some citations are intended to boost the unwarranted impact of the publication, rather than disseminate scientific advancements.
  • 4. Future Predictions
    • Predicting far into the future can be cahllending.
    As the feature-set changes with time for a paper (such as author affiliations, journal impact factor etc), The input feature on which the model is trained may not be sufficient.
  • 5. Information Extraction
    • Document level information extraction to enrich the knowledge graph is something that can be pursued but is very expensive computationally.
  • 6. Prediction in the near Future
    • Prediction of citaions in the near future tends to be exaggerated.
  • 7. Model suitability
    • Some models are more suited than others.
